# 测试说明

## 自动测试

Pytest 通过 `RUNTIME_DIR` 自动使用独立临时目录，不写入比赛演示数据库。

- 后端：输入驱动画像抽取、三类画像差异、文本切块、语义检索/Hashing 回退、动态 Review、三类评分方法。
- 后端集成：画像 → 检索 → 七状态编排 → 七类资源 → RAG → 测评 → 画像回写 → Dashboard 数据源。
- 学习证据：逐题得分写入知识点掌握度；画像、资源、答疑、测评、反馈形成学习事件。
- 展示能力：资源包包含 HTML 动态流程图，PPTX 接口返回有效 Office 文件。
- 模型验收：五种 Provider 配置映射、Mock 模式和配置缺失时的显式 fallback。
- 前端：统一 API 响应解包。
- 构建：TypeScript 编译、Vite 生产构建。
- E2E：启动前后端后，在 `frontend` 执行 `npm.cmd run test:e2e`，复用系统 Microsoft Edge。

## 人工验收

1. 无 Key 启动，状态显示 Mock。
2. 输入自然语言并生成八维画像。
3. 确认内置 Python 课程大纲、核心语法、工程实践和数据分析资料已自动入库。
4. 生成学习路径、讲义、思维导图、题目、代码、PPT 大纲和轻量多模态资源，进度从 0 到 100。
5. 答疑展示至少一个引用片段。
6. 完成四道题，评估报告写回画像。
7. 上传错误格式时收到明确提示。
8. 在资源页提交“有帮助/需要调整”，检查 Dashboard 反馈统计与学习轨迹。
9. 知识库页确认当前模式为“语义向量检索”；删除/禁用模型时应自动显示 Hashing 回退。
10. 在 PPT 大纲 Tab 下载文件，使用 PowerPoint/WPS 打开并检查六类页面。
11. 调用 `/api/config/llm-test`；真实模型成功时确认 `tested_real_model=true`、`fallback_used=false`。

当前自动测试不执行用户代码；代码题仅验证关键语句检查和人工核验标记。
