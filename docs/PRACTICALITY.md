# 实用性设计说明

本轮没有继续增加展示性资源类型，而是解决“资源生成后学生接下来做什么”的问题。

## 公开资料启发

- [OpenMAIC](https://github.com/THU-MAIC/OpenMAIC) 将持久化 Quiz 状态、项目里程碑和学习进度作为真实学习场景的重要能力。
- Roediger 与 Karpicke 的[检索练习研究](https://www.science.org/doi/10.1126/science.1127335)表明，主动回忆与测验比重复阅读更有利于长期保持。
- UNESCO 的[生成式 AI 教育指南](https://unesdoc.unesco.org/ark:/48223/pf0000386693)强调人本、透明和可监督使用，系统不应把模型生成本身当作学习完成。

## 本项目的落地方式

每次资源生成后，系统自动建立五项可执行任务：

1. 阅读个性化讲义；
2. 查看并复述思维导图；
3. 运行和修改代码案例；
4. 完成针对性练习；
5. 提交测评并复盘错题。

任务时长会根据学生风格调整：基础薄弱型增加讲解与练习时间，实践导向型增加代码时间，考试复习型增加测验时间。

为减少真实使用中的操作成本：

- 进入 Resource Center 时自动恢复最近学习的资源包，不再显示空白内容；
- “打开内容”和“标记完成”分离，避免误触导致虚假进度；
- Dashboard 的“继续学习”会定位到对应讲义、导图、代码或测评；
- 提交测评后，“针对性练习”和“测评复盘”任务依据真实提交行为自动完成。

## 真实性边界

- 勾选状态、完成时间、剩余时间和学习轨迹真实保存到 SQLite。
- 系统记录的是学习行为进度，不声称仅凭勾选就证明学生已经掌握知识。
- 最终掌握度仍以测评证据为主，当前算法属于 MVP，不是 DKT/BKT。
