# Python 程序设计基础

## 课程目标

Python 是一门强调可读性和开发效率的通用编程语言。本课程要求学习者理解“数据—操作—结果”的程序结构，能够把实际问题拆分为可测试的函数，并逐步形成调试习惯。

## 基本数据类型

常用不可变类型包括整数、浮点数、布尔值、字符串和元组；常用可变类型包括列表、字典和集合。变量保存的是对象引用，因此复制嵌套容器时需要区分浅拷贝和深拷贝。

```python
student = {
    "name": "小舟",
    "scores": [82, 91, 88],
}
average = sum(student["scores"]) / len(student["scores"])
```

列表适合保存有顺序的一组同类或相关元素，字典适合通过具有业务含义的键组织属性。集合用于去重和集合运算。

## 条件与循环

条件语句根据布尔表达式选择执行路径。循环用于处理重复任务。循环应保持目标单一；当循环体过长或重复出现时，应提取为函数。

列表推导式适合表达简单映射和过滤：

```python
passed = [score for score in scores if score >= 60]
```

当逻辑包含多层条件、异常处理或副作用时，普通 `for` 循环通常更易读。

## 函数设计

函数通过参数接收输入，通过返回值提供输出。推荐：

1. 函数名称表达动作或结果。
2. 一个函数只承担一个主要职责。
3. 避免依赖可变全局变量。
4. 使用类型标注说明输入输出。
5. 用最小样例覆盖正常、边界和错误情况。

```python
def safe_mean(numbers: list[float]) -> float | None:
    if not numbers:
        return None
    return sum(numbers) / len(numbers)
```

不要使用列表或字典作为可变默认参数。应使用 `None`，并在函数内部创建新对象。

## 异常与调试

异常表示程序运行时遇到无法按正常路径处理的情况。捕获异常时应尽量具体，避免无条件捕获所有异常并静默忽略。

```python
try:
    age = int(raw_age)
except ValueError:
    print("年龄必须是整数")
```

调试步骤：稳定复现问题、缩小范围、检查类型和边界、阅读堆栈最后一行、构造最小样例、修复后增加回归测试。
