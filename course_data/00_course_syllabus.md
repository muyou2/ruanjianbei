# Python 程序设计与数据分析：课程大纲

## 课程定位

本课程面向高校计算机及相关专业低年级学生，目标是让学习者能够使用 Python 完成从问题拆解、程序设计到数据分析项目交付的完整过程。课程资源按“先备知识—核心概念—代码实践—形成性测评—项目迁移”组织。

## 学习成果

完成课程后，学习者应能够：

1. 使用变量、分支、循环和常用容器表达基础算法。
2. 设计带参数、返回值和异常处理的函数。
3. 使用类与模块组织中小型 Python 程序。
4. 读取文件并处理常见数据格式。
5. 使用 NumPy 与 Pandas 完成数据检查、清洗、聚合和分析。
6. 对分析过程进行可复现记录，并区分数据证据、推断与限制。

## 模块与先修关系

| 模块 | 核心知识点 | 先修要求 | 建议成果 |
| --- | --- | --- | --- |
| M1 Python 入门 | 变量、类型、输入输出、运算符 | 无 | 命令行小程序 |
| M2 控制结构 | 条件、循环、推导式 | M1 | 数据筛选程序 |
| M3 容器与函数 | 列表、字典、参数、返回值、作用域 | M1-M2 | 可复用工具函数 |
| M4 面向对象与异常 | 类、对象、继承、异常处理 | M3 | 小型业务模型 |
| M5 文件与模块 | 文件读写、CSV、模块、包 | M3-M4 | 数据导入工具 |
| M6 NumPy | 数组、索引、形状、广播 | M3 | 数值计算练习 |
| M7 Pandas | Series、DataFrame、缺失值、分组 | M5-M6 | 数据清洗报告 |
| M8 综合项目 | 问题定义、探索分析、可视化、结论限制 | M1-M7 | 可复现数据分析项目 |

## 动态学习路径规则

- 掌握度低于 60%：回到概念讲解与最小代码示例。
- 掌握度 60%～84%：进行变式练习与错题复盘。
- 掌握度达到 85%：进入综合任务或跨场景迁移。
- 学习路径调整必须参考真实测评和资源反馈，不能只依赖静态画像。

## 评价方式

系统当前支持选择题、判断题的精确判分，简答题的关键词覆盖评分，以及代码题的关键语句检查。代码题不会在服务器直接执行；运行正确性、边界条件和代码质量仍需安全沙箱或人工核验。
