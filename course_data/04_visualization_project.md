# 数据可视化与综合项目规范

## 可视化的目的

图表用于回答问题，而不是装饰页面。选择图表前应明确变量类型、比较对象和希望读者看到的关系。

- 类别比较可优先考虑条形图。
- 时间变化可使用折线图。
- 两个数值变量的关系可使用散点图。
- 分布比较可使用直方图或箱线图。

坐标轴、单位、样本范围和缺失值处理必须说明。截断坐标轴可能夸大差异，应谨慎使用。

## Matplotlib 示例

```python
import matplotlib.pyplot as plt

summary.plot(kind="bar", x="topic", y="score", legend=False)
plt.ylabel("平均成绩")
plt.title("不同学习主题的平均成绩")
plt.tight_layout()
plt.show()
```

中文显示依赖本地字体配置。若图表乱码，应检查字体，而不是修改原始数据。

## 综合项目交付物

一个合格的数据分析项目至少包含：

1. 问题定义与数据来源。
2. 字段字典和质量检查结果。
3. 可复现的数据清洗代码。
4. 至少两种与问题匹配的分析或图表。
5. 结论、证据和适用范围。
6. 局限性及下一步验证计划。

## 项目评价量规

| 维度 | 达标标准 |
| --- | --- |
| 正确性 | 代码可复现，统计口径一致 |
| 可解释性 | 每个清洗与分析步骤有理由 |
| 证据性 | 结论能够追溯到数据或课程资料 |
| 工程性 | 文件结构清晰，避免硬编码路径 |
| 反思性 | 主动说明偏差、缺失和不确定性 |

项目练习应根据学习者画像调整：基础薄弱型拆成小步骤，实践导向型增加真实数据任务，考试复习型强化 API 辨析和典型易错点。
